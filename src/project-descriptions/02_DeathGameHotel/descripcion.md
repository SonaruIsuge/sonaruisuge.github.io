# Death Game Hotel

## 概要

Death Game Hotel は、プレイヤーが財産だけでなく身体の一部までも賭けて勝負する、オンラインマルチプレイヤー対応のVRカードゲームです。一瞬の判断が勝敗を左右する、緊張感のあるゲーム体験が特徴です。

--- 
<br>

本プロジェクトはインターン期間中に参加した2つ目の案件であり、参加時点ではすでに開発中盤に入っていました。そのため、既存システムを短期間で理解しながら機能追加や改善を行うことが求められました。

主にVRインタラクティブオブジェクトの実装、既存VR UIシステムの改善・リファクタリング、UIおよびフォント関連のパフォーマンス改善を担当しました。本プロジェクトを通じて、商業ゲーム開発のワークフローに加え、VR開発およびマルチプレイヤー同期の知識と経験を身につけました。

## 技術的な取り組み

### マルチプレイヤー環境におけるインタラクティブオブジェクト開発

最初の担当業務は、ゲーム内で使用される複数のインタラクティブオブジェクトの実装でした。

基本的なインタラクションフレームワークは既に構築されていたため、各オブジェクト固有の挙動を実装しました。

開発中、一部のオブジェクトがマルチプレイヤー環境で正しく同期されない問題が発生しました。Photon Fusionの同期仕様を調査し、先輩エンジニアとも相談しながら原因を分析した結果、RPC を利用した同期処理を導入することで問題を解決しました。

この経験を通して、ネットワーク同期や権限管理の仕組みについて理解を深めることができました。

### VR UIシステムのリファクタリング

本プロジェクトで最も大きな業務の一つが、既存VR UIシステムのリファクタリングでした。

このUIはルーム入室、チュートリアル、ショップなど多くの機能を担っており、長期間にわたり複数の開発者によって改修されてきたため、構造が複雑化していました。

まず画面間の遷移や依存関係を整理・可視化し、不要なオブジェクトや処理を削除しました。その後、Hierarchy構成を見直し、共通処理をHelper Scriptとして切り出すことで重複コードを削減しました。

さらに調査を進める中で、他システムから不要なUI更新が頻繁に行われていることを発見し、処理を見直すことで描画負荷も改善しました。

結果として、UI全体の保守性・可読性・パフォーマンスを向上させることができました。

### Dynamic Fontを用いたフォント最適化

本作では日本語テキストを大量に使用していたため、TextMeshProのフォントアトラスが非常に大きくなり、メモリ使用量やビルドサイズの増加が課題となっていました。

そこで公式ドキュメントを調査し、Dynamic Font Asset の仕組みを検証・実装しました。

検証の結果、必要な文字だけを動的に生成する方式へ変更することで、フォントアセットの容量を削減しながら表示品質を維持できることを確認しました。

自ら技術調査を行い、ドキュメントから得た知識を実際のプロジェクトへ導入し、その効果を検証した上でチームへ還元できたことは、特に印象に残っている経験です。
