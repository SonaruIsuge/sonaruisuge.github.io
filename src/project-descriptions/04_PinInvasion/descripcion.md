# Pin Invasion

## 概要

Pin Invasion は、ピンボールとサバイバー系ゲームを組み合わせた実験的なアクションゲームです。プレイヤーはボールを何度も打ち返して敵を倒し、経験値を獲得してレベルアップしながら様々な強化を選択し、最終的にボスへ挑みます。

--- 
<br>

本プロジェクトは、社内で進められていた新規ゲーム企画のプロトタイプとして開発が始まりました。最初は「ピンボールを使ったゲーム」という漠然としたアイデアからスタートし、何度も企画の議論や試作を重ねた結果、「ピンボール × サバイバーゲーム」というコンセプトへ発展しました。

少人数で開発を行っていたこともあり、私はゲーム全体の基盤となるシステム設計から実装まで幅広く担当しました。また、設計方針について先輩エンジニアと何度も議論を重ねたことで、保守性・拡張性を考慮したゲームシステム設計や、長期的な開発を見据えたアーキテクチャについて多くを学ぶことができました。

## 技術的な取り組み

### ゲームフローとシステムアーキテクチャ

ゲーム全体の進行を管理するため、Singleton パターンを採用した Game Manager を設計し、ゲーム全体で一貫した状態管理を行いました。

ゲーム進行には State Machine を採用し、「タイトル」「プレイ中」「リザルト」など、それぞれの状態が自身のゲームロジックとUI更新を担当する構成としました。これにより、ゲームフローが明確になり、新しい状態を追加する場合でも既存システムへの影響を最小限に抑えられるよう設計しています。

さらに、システム間の依存関係を減らすため、ゲーム全体でイベント駆動アーキテクチャを積極的に採用しました。各システムは必要なイベントのみを購読する構成とし、互いを直接参照しないことで、高い保守性と拡張性を実現しています。

レベル構成については Multiple Scene を利用し、ゲームロジックと背景・演出などのアートアセットを分離しました。これにより、新しいステージを追加する際の作業を簡素化するとともに、必要なシーンのみを読み込むことで実行時の負荷も軽減しています。

本システムでは、一貫して「疎結合」「拡張性」「保守性」を重視したアーキテクチャを目指しました。

### 敵システム・Waveシステム

敵システムでは、継承中心ではなく Composition を重視した設計を採用しました。

敵の基底クラスには最低限の共通処理と Virtual メソッドのみを実装し、移動や攻撃などの振る舞いはコンポーネントとして分離しました。これにより、新しい敵を実装する際には必要なコンポーネントを組み合わせるだけで、多様な敵を容易に作成できる構成となっています。

敵生成システムでは Object Pool を利用し、実行時の生成・破棄を抑えることでGC負荷を削減しました。また、敵の出現位置は Interface を介して管理することで、ランダム生成や固定位置生成など、スポーンアルゴリズムを柔軟に切り替えられる設計としています。

Wave管理については ScriptableObject を利用した Data-driven Design を採用しました。Waveデータをコードから切り離すことで、ゲームバランスの調整を容易にしています。

さらに UniTask を利用してWaveの進行を非同期で管理し、開発中には任意のWaveからゲームを開始できるデバッグ機能も実装しました。これにより、終盤Waveの検証やバランス調整を効率的に行えるようになりました。

### アップグレードシステム

レベルアップ時に表示されるアップグレードシステムも設計・実装しました。

Upgrade Managerでは、ゲーム内に存在するすべてのアップグレード情報を管理し、現在の取得状況をもとに条件を満たす候補のみを動的に抽出する仕組みを構築しました。

また、各スキルは共通の抽象基底クラスを継承する構成とし、発動処理はすべて統一されたインターフェースから実行できるよう設計しています。

この設計により、新しいスキルやアップグレードを追加する際にも既存コードへの影響を最小限に抑えることができ、機能追加やバランス調整を容易に行える拡張性の高いシステムとなりました。
