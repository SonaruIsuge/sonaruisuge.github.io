# Grid Framework

## 概要

本プロジェクトは、グリッドベースのゲームシステムを研究・検証することを目的とした個人プロジェクトです。特定のゲームジャンルを想定するのではなく、再利用可能なグリッドアーキテクチャを基盤として、経路探索や建築システムなど、さまざまなゲームシステムへ応用できるフレームワークの構築を目指しています。

開発は業務外の時間を利用して進めており、ゲームプレイの実装よりも、将来的に再利用できるシステム設計を重視しています。現在は、汎用的なグリッドシステムを中心に、最適化した A* 経路探索、非同期パス探索、NPC ナビゲーション、建築システムなどを実装しています。

まだ開発初期段階ではありますが、商業開発では試しにくいアーキテクチャ設計やアルゴリズム最適化を検証するための実験的なプロジェクトとして継続的に開発を行っています。

## 技術的な取り組み

### Generic Grid Framework

> **ゲームロジックとグリッド管理を分離した汎用的なグリッドフレームワークを設計し、経路探索・建築システム・将来的なゲームシステムが共通の基盤を利用できる構成を実現しました。**

本プロジェクトは、ゲーム固有の処理から独立した汎用的なグリッドシステムを基盤として設計しています。

各セルにはゲーム専用のデータを固定で保持するのではなく、ジェネリクスを利用することで任意のデータ型を保持できるようにしました。これにより、一つのグリッド実装を経路探索や建築システムなど、異なるゲームシステム間で再利用できます。

また、ワールド座標とグリッド座標の相互変換機能を提供し、ゲーム内オブジェクトやプレイヤー入力をグリッド上の論理座標へ容易に変換できるようにしています。

さらに、セルの内容が更新された際には Delegate を通じて変更通知を行うイベント駆動型の仕組みを採用しました。これにより、各システムがグリッドへ直接依存することなく変更を受け取ることができ、システム間の結合度を抑えた設計となっています。

### Optimized A* Pathfinding

> **Heap による探索最適化、マルチスレッドによる非同期処理、地形コスト、経路簡略化を組み合わせた、高効率な A\* 経路探索システムを実装しました。**

経路探索には、汎用グリッドシステム上で動作する A* アルゴリズムを採用しています。

探索効率を向上させるため、Open List は線形探索ではなく Binary Heap を利用して実装しました。これにより、最小コストノードの取得処理を O(N) から O(log N) へ改善し、大規模マップにおける探索性能を向上させています。

また、経路探索処理はバックグラウンドスレッド上で非同期に実行しています。Unity のメインスレッドをブロックすることなく探索を行い、完了後はスレッドセーフな方法で結果をメインスレッドへ返却する構成としました。そのため、複数の NPC が同時に経路探索を要求した場合でも、フレームレートへの影響を最小限に抑えられます。

さらに、探索後の経路については、一直線上に並ぶ不要な中継ノードを幾何学的に削除し、よりシンプルな経路へ最適化しています。

加えて、各グリッドセルには移動コストを設定できるようにしており、異なる地形を表現できます。NPC はコストを考慮した経路を選択するため、追加のゲームロジックを書くことなく自然な経路選択を実現しています。

### NPC Navigation

> **UniTask を利用した非同期移動と幾何学的なウェイポイント判定により、経路変更へ柔軟に対応できる滑らかな NPC ナビゲーションを実装しました。**

NPC の移動処理は、経路探索システムとは独立した構成で設計しています。

移動更新には UniTask を利用しており、新しい目的地が指定された際には現在の移動タスクを容易にキャンセルし、新しい経路へ即座に切り替えられるようにしました。これにより、複数の移動処理が同時に実行されることを防ぎつつ、シンプルな非同期制御を実現しています。

また、Waypoint の切り替えは距離判定だけではなく、現在の移動方向とウェイポイントの垂線を利用した幾何学的な判定を採用しました。これにより、NPC は急停止や細かな往復動作を起こすことなく、自然な軌道で曲がることができます。

経路探索と移動処理を分離したことで、将来的に別の経路探索アルゴリズムへ変更した場合でも、移動システム側を修正することなく利用できる構成となっています。

### Grid-based Building System

> **汎用グリッドフレームワークを基盤として、多層配置・建築物の回転・柔軟なグリッド操作に対応した拡張性の高い建築システムを実装しました。**

建築システムは、グリッドフレームワークの上に構築された独立したシステムとして実装しています。

マウス座標をグリッド座標へ変換することで、カメラ位置やワールド座標に依存せず、建築物を正確にセルへスナップできるようにしました。

建築前には 90 度単位で建築物を回転できる機能を備え、グリッドとの整合性を保ちながら自由度の高い配置を実現しています。

また、一つのセルに単一オブジェクトのみを保持するのではなく、複数のレイヤーを管理できる構造を採用しました。これにより、異なる種類のオブジェクトを同じセルへ配置でき、より柔軟な建築システムを実現しています。

削除機能についてもレイヤー単位で管理しており、指定したレイヤーのオブジェクトのみを削除できるため、他のオブジェクトへ影響を与えることなく編集を行えます。

建築システムをゲーム固有の機能として設計するのではなく、グリッドフレームワークを利用する一つのモジュールとして実装することで、将来的にはシミュレーションゲームやストラテジーゲームなど、さまざまなジャンルへ応用可能な構成としています。
